1. Présentation du jeu Sokoban

Imagine un entrepôt rempli de caisses éparpillées. Tu es le magasinier. Ta mission : pousser chaque caisse jusqu'à son emplacement marqué au sol. Facile en apparence — mais attention, tu ne peux que pousser, jamais tirer. Et si une caisse se retrouve coincée dans un angle, la partie est fichue.

Ce casse-tête s'appelle le Sokoban (倉庫番 en japonais, soit « gardien d'entrepôt »). Il a été inventé en 1982 par Hiroyuki Imabayashi. Aujourd'hui encore, il fascine les chercheurs en informatique : certains niveaux ont été mathématiquement prouvés comme appartenant aux problèmes les plus difficiles qui existent.

Est-ce que ça t'intéresse d'en construire un toi-même, de A à Z, en Python ? C'est exactement ce que tu vas faire dans ce projet.

1.2 Les règles

Règles fondamentales
  • Le joueur se déplace dans 4 directions : haut, bas, gauche, droite.
  • Si une caisse se trouve devant lui, il la pousse — à condition que la case derrière la caisse soit libre.
  • Impossible de pousser deux caisses à la fois.
  • La partie est gagnée quand toutes les caisses reposent sur une cible (plus aucun $ dans la grille).

2. Les classes Python du Sokoban

Avant de coder, arrêtons-nous une seconde. Dans un entrepôt réel, tu as des objets bien distincts : le plan de la salle, les caisses, le magasinier, les emplacements marqués au sol. En Python, on modélise chacun par une classe. C'est le principe fondamental de la programmation orientée objet.

2.1 La classe Niveau

Le niveau est le plan de l'entrepôt. Il contient la grille (une liste de listes de caractères), les dimensions, et la liste des positions cibles.

Cours — Classe, constructeur, attributs

En Python, une classe se définit avec class. Le constructeur __init__ est appelé à la création de l'objet. self désigne l'instance courante. Un attribut est une variable propre à l'objet : self.grille, self.nb_lignes…

class Niveau:
    def __init__(self, grille_texte):
        # grille_texte : liste de chaînes de caractères
        self.grille    = [list(ligne) for ligne in grille_texte]
        self.nb_lignes = len(self.grille)
        self.nb_cols   = max(len(r) for r in self.grille)
        self.cibles    = self._trouver_cibles()

    def _trouver_cibles(self):
        """Retourne la liste des positions (ligne, col) des cibles."""
        positions = []
        for l in range(self.nb_lignes):
            for c in range(len(self.grille[l])):
                if self.grille[l][c] in ('.', '*', '+'):
                    positions.append((l, c))
        return positions

2.4 Héritage : NiveauChrono

Cours — Héritage et spécialisation

L'héritage permet à une classe « fille » de réutiliser et d'enrichir une classe « mère ». La classe fille hérite automatiquement de tous les attributs et méthodes de la mère. Elle peut en ajouter de nouveaux, ou en redéfinir (surcharger) certains.

class Animal:
    def __init__(self, nom):
        self.nom = nom
    def crier(self):
        return "..."

class Chien(Animal):          # Chien hérite de Animal
    def crier(self):          # surcharge de la méthode
        return "Ouaf !"

rex = Chien("Rex")
print(rex.nom)    # "Rex"    → hérité de Animal
print(rex.crier())# "Ouaf !" → redéfini dans Chien

En Sokoban, on peut créer NiveauChrono qui hérite de Niveau et ajoute un chronomètre.

2.5 Récursivité : compter les caisses placées

Cours — Récursivité et pile d'appels

Une fonction récursive s'appelle elle-même. Elle comporte toujours :

  • un cas de base qui arrête la récursion,
  • un appel récursif qui se rapproche du cas de base.

À chaque appel récursif, Python empile un nouveau cadre d'appel sur la pile d'exécution. Quand le cas de base est atteint, les cadres se dépilent un à un.

Pile d'appels pour nb_caisses(g, 0)

nb_caisses(g, 0) ← sommet
nb_caisses(g, 1)
nb_caisses(g, 2)
fond (cas de base : idx == len(g))

Les appels s'empilent (↑) jusqu'au cas de base,
puis les résultats se combinent en redescendant (↓).

2.6 Challenge — Historique des coups avec une pile

Cours — La structure pile (LIFO)

Une pile (en anglais stack) fonctionne comme une pile d'assiettes : on ajoute et on retire toujours par le dessus. C'est le principe LIFO — Last In, First Out. En Python, une liste suffit :

pile = []
pile.append("état_1")   # empiler
pile.append("état_2")   # empiler
dernier = pile.pop()     # dépiler → "état_2"

Dans un Sokoban, on stocke l'état de la grille avant chaque coup. Pour annuler, on dépile le dernier état.

4. Le gameplay — Graphes et algorithmes de parcours

Est-ce que le joueur peut atteindre la case (3, 5) depuis sa position actuelle ? Existe-t-il un chemin pour pousser une caisse jusqu'à sa cible ? Pour répondre à ces questions, on va voir le plateau comme un graphe.

4.1 Le plateau modélisé comme un graphe

Cours — Graphe : sommets, arêtes, représentation

Un graphe est un ensemble de sommets reliés par des arêtes. Dans notre Sokoban :

  • chaque case libre est un sommet,
  • deux cases adjacentes accessibles sont reliées par une arête.

On peut représenter ce graphe sous forme d'une liste d'adjacence : un dictionnaire où chaque clé est un sommet et la valeur est la liste de ses voisins.

graphe = {
    (1,1): [(1,2), (2,1)],
    (1,2): [(1,1), (1,3)],
    ...
}

4.2 Parcours en largeur — BFS

Cours — BFS avec une file (FIFO)

Le parcours en largeur (BFS — Breadth First Search) explore le graphe couche par couche, en commençant par les voisins les plus proches. Il utilise une file (FIFO — First In, First Out) :

from collections import deque
file = deque([depart])   # initialise avec le sommet de départ
while file:
    sommet = file.popleft()  # dépile par la gauche
    for v in voisins(...):
        if v not in visite:
            file.append(v)   # empile à droite

Le BFS garantit de trouver le chemin le plus court dans un graphe non pondéré.

Indice 2 — Cas particulier

4.3 Parcours en profondeur — DFS

Cours — DFS avec une pile ou récursivité

Le parcours en profondeur (DFS — Depth First Search) s'enfonce dans le graphe le plus loin possible avant de revenir en arrière. On peut l'implémenter :

  • avec une pile explicite (version itérative),
  • ou de façon récursive (la pile d'appels joue le rôle de la pile).
# Version itérative avec pile
def dfs(grille, depart):
    visite = set()
    pile   = [depart]
    while pile:
        pos = pile.pop()   # dépile par le dessus (LIFO)
        if pos in visite: continue
        visite.add(pos)
        for v in voisins(grille, pos[0], pos[1]):
            if v not in visite:
                pile.append(v)
    return visite

4.4 Programmation dynamique — Distance optimale

Cours — Programmation dynamique et mémoïsation

La programmation dynamique consiste à décomposer un problème en sous-problèmes, les résoudre une seule fois, et mémoriser les résultats dans un cache pour ne pas les recalculer. C'est la mémoïsation.

Exemple classique — rendu de monnaie :
Pour rendre 6€ avec des pièces de [1, 2, 5], quelle est le nombre minimal de pièces ?

cache = {}

def rendu(somme, pieces):
    if somme == 0: return 0
    if somme in cache: return cache[somme]  # déjà calculé !
    mini = float('inf')
    for p in pieces:
        if p <= somme:
            mini = min(mini, 1 + rendu(somme - p, pieces))
    cache[somme] = mini
    return mini

# rendu(6, [1,2,5]) = 2  (5+1)

En Sokoban, la distance minimale entre deux cases se calcule de même façon : on mémorise les distances déjà calculées dans un dictionnaire.
